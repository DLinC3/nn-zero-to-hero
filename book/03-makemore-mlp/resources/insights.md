# Makemore MLP — Review Insights

- A complete n-gram table needs `V^T` context rows, so discrete long-context counts become sparse exponentially.
- Shared embeddings let statistically similar characters and contexts reuse what each example learns.
- An embedding row is learned for predictive usefulness, not assigned a predefined linguistic meaning.
- `C[index]` is exactly `one_hot(index) @ C` with the zero multiplications removed.
- A name of length `L` still yields `L + 1` targets because the model learns continuation and termination.
- Build train/dev/test from complete names before constructing character examples.
- The MLP shape contract is `(N,T) → (N,T,D) → (N,T·D) → (N,H) → (N,V)`.
- Track axis semantics as well as shape because a semantically wrong permutation can remain shape-compatible.
- `flatten(1)` preserves the batch axis and concatenates all context features.
- A compatible tensor view changes metadata without copying storage; concatenation allocates new storage.
- Bias `(H,)` broadcasts across the batch dimension of `(N,H)`.
- Raw logits are unconstrained scores; softmax converts them into next-character probabilities.
- Cross-entropy is the stable, fused implementation of log-softmax plus target NLL.
- Subtracting the largest logit preserves softmax probabilities while preventing exponential overflow.
- Overfitting a tiny batch is a high-value test of data, targets, forward, backward, and updates.
- Tiny-batch loss need not reach zero when the same context legitimately has several next characters.
- A mini-batch gradient is a noisy estimate of the full gradient, not an unrelated direction.
- Cheap approximate gradients can make more progress per second than expensive exact gradients.
- Search learning rates on a logarithmic scale because useful magnitudes span powers of ten.
- A late learning-rate decay trades coarse progress for smaller settling steps.
- Train optimizes parameters; dev guides hyperparameters; test audits the frozen process.
- Repeated test inspection trains the experimenter on the test set even without backpropagation.
- High and similar train/dev loss suggests underfitting or optimization limits, not good generalization by itself.
- Falling train loss with stalled dev loss signals that added capacity needs regularization.
- Increasing context raises the information ceiling; width cannot recover characters absent from the input.
- Increasing embedding dimension removes a representation bottleneck before hidden width can be fully useful.
- Learned 2-D vowel clustering reflects predictive interchangeability, not an intrinsic semantic coordinate system.
- A wider model becomes useful only when initialization, optimizer, schedule, and regularization train it effectively.
- Generation reuses the training shape contract with batch size one and feeds sampled tokens back into context.
- `model.eval()` changes dropout behavior; inference mode disables autograd; neither substitutes for the other.
